## What Went Well

### Test Coverage Discipline
Test count grew steadily from 42 to 222 with 100% green at every story boundary. No tests were skipped or marked flaky. The framer-motion mock pattern established in Story 1.3 was reused consistently through Story 1.8 without regression.

### Lighthouse CI as Safety Net
Adding LHCI in Story 1.7.2 caught a color contrast failure (`#A855F7` in light mode) that axe-in-jsdom cannot detect due to CSS computation limitations. Without LHCI the bug would have reached production.

### ESLint Zero-Error Discipline
The ESLint flat config with strict rules (import/order, no-relative-parent-imports, react-refresh) established in Story 1.1 paid dividends — 0 lint errors across all 9 stories. Catching violations early prevented accumulation.

### Spring Animation Token System
`SPRING_GENTLE` (hover/reveal), `SPRING_SNAPPY` (navigation), `SPRING_BOUNCY` (celebrations) — clear semantic mapping made token usage consistent. `useMotion` hook with built-in `prefers-reduced-motion` fallback is a reusable pattern for future epics.

### Story Delivery Cadence
9/9 stories delivered, no rollover, no scope cuts. i18n (Story 1.6) and accessibility (1.7.1, 1.7.2) delivered on schedule without needing buffer epics.

- Story 1.4: UTC timezone bug in date formatting caught in code review after implementation

- Story 1.6: persist key `'language-preference'` implemented vs `'pf_language'` in spec (unresolved debt)

### No E2E Test Coverage
222 unit/component tests, but zero integration or E2E tests for real user flows (navigation, language toggle, project filter). Critical paths are untested at the browser level.

## Key Learnings (for MEMORY.md / future epics)

### Library Quirks
- `vitest-axe@0.1.0`: Do NOT import `extend-expect`. Use `import { toHaveNoViolations } from 'vitest-axe/matchers'` + `expect.extend(...)`.
- `vi.doMock` after `vi.mock` has no effect on already-imported modules. Use `vi.fn().mockReturnValue()` + `vi.mocked()` per-test instead.
- `axe` in jsdom cannot verify CSS contrast. Always gate with LHCI.

### Architecture Patterns
- `AppRoutes` as inner component inside `<BrowserRouter>` — required for `useLocation()` in page transition logic.
- `AnimatePresence mode="wait"` with `<Routes location={location} key={location.pathname}>` for page transitions.
- `useMotion` hook centralizes `useReducedMotion` — single source of truth for animation enablement.

### Testing Patterns
- framer-motion mock: `vi.mock('framer-motion', () => ({ motion: { div: ..., article: ... }, AnimatePresence: ... }))` — must include all motion elements used in component.
- `getByRole('link', { name: ... })` matches `aria-label`, not text content — design queries around actual accessible names.
- `role="listitem"` on `<a>` overrides implicit link role — use `ul > li > a` structure.

### Tailwind / shadcn
- Safari + VoiceOver removes list semantics with `list-none` — always add explicit `role="list"` on `<ul>`.
- Touch targets: `min-h-[44px]` on interactive elements, add regression test to prevent silent revert.
