## What Went Well

### Foundation Stories Were Clean
Stories 2.1–2.3 delivered without surprises: Spring Boot 3.5.11 + Testcontainers from day one, Flyway migration working correctly, GlobalExceptionHandler and OpenAPI live. 20/20 tests — no flaky tests, no skips.

### Testcontainers Commitment Honored
Epic 1 action item A8 (Testcontainers with real PostgreSQL, no H2) was executed exactly as committed. Story 2.2 integration tests run against a real Postgres container — solid foundation for Epic 3 data layer tests.

### Fast Pivot Decision-Making
When Oracle A1 failed, the team evaluated Fly.io, Koyeb, Railway, Render, and GCR+Neon systematically and selected the best free-tier option within a single session. GCR + Neon selection was correct — free tier is genuinely free (not "free with asterisk"), scale-to-zero fits portfolio traffic pattern, Neon PITR eliminates backup story entirely.

### Spring Boot Debug Learnings Documented Immediately
Every Spring Boot 3.5.x quirk discovered (wrong override method name, `@WebMvcTest` inner class, CSRF in POST tests) was captured in MEMORY.md in real time. None repeated.

### Architecture Adapted Gracefully
Cloud Run TLS termination made Story 2.4.1 (Nginx) entirely obsolete — not a failure, a simplification. The architecture contract was maintained (HTTPS, forward headers strategy) with less complexity.

## What Didn't Go Well

### Infrastructure Availability Not Pre-Verified
Oracle A1 "Always Free" tier was selected in architecture without verifying actual slot availability. Result: Oracle A1 full across all Asia-Pacific regions, Singapore and Tokyo confirmed in Oracle's own capacity notice. The architectural decision was made based on marketing docs, not a provisioning test.

### "Free Tier" Claims Require Direct Verification
Fly.io discontinued free tier for new accounts without clear announcement — docs still said "free" while the onboarding UI showed credit card requirement + minimum monthly charge (~$2–5/month). This is the second consecutive infrastructure assumption that was wrong.

### sa-key.json Exposed Twice
The GCP service account key file was accidentally exposed twice in the same session via VS Code IDE file selection. The key was rotated both times successfully, but:
- The same mistake happened twice, suggesting the instruction "don't open the file" is insufficient protection.
- No automated safeguard (pre-commit hook, .gitignore, git-secrets) was in place.

### Epic 1 Action Items Not Followed Through
7 of 8 Epic 1 action items were not completed. A1 (pre-impl checklist) and A2 (known-library-bugs.md) were specifically scoped for early Epic 2 — both skipped. The retro commitment loop is broken if action items have no tracking mechanism.

### PowerShell / Git Bash Environment Friction
Several `gcloud` CLI commands failed in PowerShell due to line continuation (`\`) behavior, requiring retries in Git Bash. Minor friction but repeated across multiple commands.

## Key Learnings (for MEMORY.md / future epics)

### Infrastructure & Vendor Risk
- Always provision a test instance before committing a platform in architecture. "Always Free" marketing ≠ slot availability.
- Verify vendor pricing via the actual billing/onboarding flow, not docs. Docs lag reality.
- Have a contingency platform ready when selecting free-tier infrastructure.

### Security
- Service account keys in JSON must never be opened in an IDE. Clipboard-only via `Get-Content | Set-Clipboard` or equivalent.
- Add `*sa-key*.json` to `.gitignore` immediately when creating credentials — before the file is created.
- Use `gcloud iam service-accounts keys delete` + recreate to rotate exposed keys (documented pattern confirmed to work).

### Spring Boot 3.5.x (already in MEMORY.md)
- `ResponseEntityExceptionHandler`: override `handleNoResourceFoundException` (not `handleNoResourceFound`)
- `@WebMvcTest` inner class not picked up — use separate `TestStubController.java`
- CSRF blocks POST in `@WebMvcTest` — exclude `SecurityAutoConfiguration.class`
- `NoResourceFoundException` replaces `NoHandlerFoundException` in SB 3.2+

### GCR + Neon Stack
- HikariCP: `max-pool-size=3`, `keepalive-time=30000` for Neon auto-suspend compatibility
- `gcloud` CLI: use Git Bash, not PowerShell (line continuation `\` fails in PS)
- Artifact Registry auth: `gcloud auth configure-docker asia-southeast1-docker.pkg.dev --quiet`
- Service account roles needed: `run.admin`, `artifactregistry.writer`, `iam.serviceAccountUser`
